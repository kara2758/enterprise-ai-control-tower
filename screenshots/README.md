# Screenshots

This folder holds **screenshots of the dashboard and key features**, used in the
GitHub README, documentation, and the LinkedIn case study.

> ⚠️ Every screenshot shows **synthetic data only** — no real company, customer,
> vendor, invoice, employee, or personal data.

## Recommended screenshots to capture

Run the dashboard (`streamlit run app.py`) and capture each section:

| Screenshot | What to capture | Suggested file name |
|------------|-----------------|---------------------|
| Dashboard overview | Full top-of-page view (header + KPI cards) | `dashboard_overview.png` |
| Executive KPI cards | The executive KPI metric cards | `executive_kpis.png` |
| Risk summary charts | The Plotly risk/operational charts | `risk_charts.png` |
| Top Risk Findings table | The prioritized risk register table | `top_risks_table.png` |
| AI Executive Summary | The AI summary section (after clicking Generate) | `ai_summary.png` |
| AI Manager Assistant | The assistant section with a sample answer | `manager_assistant.png` |
| Dataset preview | The expandable dataset preview | `dataset_preview.png` |
| Weekly report | The generated `reports/weekly_management_report.md` rendered | `weekly_report.png` |

## Tips

- Use clear, high-resolution captures (a wide browser window looks best).
- For the AI sections, click the buttons first so the output is visible.
- Keep the same theme across screenshots for a consistent look.
- Once captured, reference them in the README, e.g.:
  `![Dashboard overview](screenshots/dashboard_overview.png)`
