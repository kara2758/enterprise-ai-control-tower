# AI Manager Assistant — Design

This document describes the AI Manager Assistant implemented in
`src/assistant.py` (Stage 8) and integrated into the Streamlit dashboard.

> ⚠️ The assistant answers from **synthetic data** for the fictional company
> **ABC Global Services**. It is for portfolio and demonstration purposes only.

---

## 1. Purpose

The AI Manager Assistant lets executives and operations managers ask
natural-language questions about the current KPI results and risk register, and
get concise, management-ready answers. Example questions:

- What are the top operational risks?
- Which department has the most risk findings?
- What is the current financial exposure?
- Summarize SLA pressure / approval bottlenecks.
- Prepare a short briefing for Monday's management meeting.

It complements the AI Executive Summary (Stage 7): the summary is a one-shot
briefing; the assistant is interactive Q&A.

---

## 2. Demo Mode vs Live Mode

| | Demo Mode | Live Mode |
|---|-----------|-----------|
| **Trigger** | `force_demo=True`, or no `OPENAI_API_KEY` / `OPENAI_MODEL` | Both env vars present and `force_demo=False` |
| **AI call** | None — deterministic, keyword-routed | Calls the OpenAI API |
| **Output** | Built from context values | Generated by the model |
| **`mode` field** | `"demo"` | `"live"` (or `"fallback"` on error) |
| **API key needed** | No | Yes |

**Fallback:** if live mode is selected but the SDK is missing, credentials are
absent, the API call fails, or the response is empty, the assistant returns a
deterministic demo answer with `mode = "fallback"` and a clear note. It never
crashes the dashboard.

### Demo intent routing
`classify_demo_question` maps a question to one of:
`top_risks`, `financial_exposure`, `sla_pressure`, `approval_bottlenecks`,
`department_risk`, `management_briefing`, or `general_summary` — using simple
keyword matching. `answer_demo_question` then returns a deterministic answer for
that intent, referencing real context values. Intent classification is
intentionally simple; it only drives demo mode.

---

## 3. Why Only Compact KPI/Risk Context Is Used

`build_assistant_context` constructs a **compact** context containing only:

- `executive_kpis`
- invoice / ticket / approval KPI highlights
- `risk_summary`
- the **top 10** risk records
- top risk category and department

The full CSV datasets are **never** sent to the API. This minimizes data
exposure, reduces token usage and cost, and keeps answers grounded in the
metrics that matter.

---

## 4. Suggested Questions

`get_suggested_questions()` returns a ready-made list the dashboard offers via a
selectbox, e.g.:

- What are the top 5 risks right now?
- Which department needs management attention first?
- What is the current financial exposure?
- Summarize customer SLA pressure.
- Summarize approval bottlenecks.
- Prepare a 5-bullet executive briefing.
- What should management prioritize this week?

---

## 5. Answer Structure

`answer_manager_question` always returns the same dictionary shape:

```json
{
  "mode": "demo | live | fallback",
  "question": "the question asked",
  "answer": "concise, management-ready answer text",
  "confidence_note": "..."
}
```

An empty question returns a friendly prompt asking the user to enter one.

---

## 6. Dashboard Integration

The dashboard adds an **AI Manager Assistant** section after the AI Executive
Summary (and before the charts):

- A short explanation.
- A **"Use demo mode for assistant"** checkbox (default: True).
- A **selectbox** of suggested questions.
- A **text input** for a custom question.
- An **"Ask Assistant"** button.

The answer is generated **only when the button is clicked** — never
automatically on page load. The mode (demo / live / fallback) is shown clearly,
followed by the question, answer, and confidence note.

---

## 7. Safety & Privacy Rules

- API keys are read from the environment only; never hardcoded, printed, logged,
  or returned.
- `.env` is git-ignored; only `.env.example` (placeholders) is committed.
- Only aggregated KPIs, the risk summary, and top-risk records are sent to the
  API.
- No external API is called at import time; live mode runs only on explicit
  invocation (the dashboard button or a direct function call).
- Fallback messages reference only the error *type*, never raw exception
  messages that could leak sensitive details.

---

## 8. Limitations

- Demo answers are template-based and keyword-routed — they will not handle every
  phrasing as gracefully as live mode.
- The assistant only knows the compact context it is given; it cannot answer
  questions about data outside the KPIs/risks (and is instructed to say so in
  live mode).
- It is stateless — there is no multi-turn memory in this stage.

---

## 9. Future Improvements

- **RAG over documents (later stage):** ground answers in uploaded policy /
  contract / report documents.
- **n8n automation (Stage 9):** trigger workflows (alerts / scheduled reports)
  from assistant outputs or risk findings.

These are intentionally **out of scope** for Stage 8, which focuses solely on
management Q&A over existing KPI and risk outputs.
